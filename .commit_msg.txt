feat: add vulnerability-injection deep-dive phase after A-class convergence

## Overview

After Phase 2 A-class vocabulary alignment converges, the system now enters
a deep-dive phase that feeds previously-discovered B-class vulnerability
patterns back into sub-agent prompts, enabling targeted search for similar
security issues across all workflows.

## New two-stage Phase 2 convergence

- Stage 1 - Vocabulary alignment (existing behavior):
  A-class convergence via zero-diffs, delta-stabilization, or oscillation
  detection.  Instead of terminating, this now transitions to Stage 2.

- Stage 2 - Deep-dive (new):
  - Activates deepdive_active=True in state
  - Previously discovered B-class diffs are classified into reusable
    vulnerability pattern categories (peer_penalty_divergence,
    missing_safety_guard, timeout_rejection_divergence,
    fork_choice_divergence)
  - Patterns are injected into both Phase 2 Main and Sub prompts
  - Converges when B-class count is stable for DEEPDIVE_STABLE_WINDOW
    iterations or MAX_ITER_DEEPDIVE rounds reached
